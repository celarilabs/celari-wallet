# Celari Wallet

## Project Structure
- `extension/` — Chrome extension + offscreen.js (Aztec PXE bundle)
- `ios/CelariWallet/` — SwiftUI iOS app (XcodeGen project)
- `contracts/` — Noir smart contracts (celari_passkey_account, celari_recoverable_account)
- `sdk/` — TypeScript SDK
- `scripts/` — Deploy scripts
- `bridge/` — L1↔L2 token bridge (Solidity + Noir + SDK)
- `fork/` — Custom forks (noir_rs, swoirenberg)
- `docs/` — Architecture docs, specs, plans

## Build Pipeline
- **offscreen.js**: `node extension/build.mjs` → 3-pass esbuild (standard → Aztec bundle → iOS variant)
- **iOS**: `xcodegen generate` (if new Swift files) → `xcodebuild` → `xcrun devicectl install`
- **Contracts**: `aztec compile` → `aztec codegen`
- **SDK**: `cd sdk && npx tsc`
- **Bridge L1**: `cd bridge/contracts/l1 && forge build`

## Key Architecture Decisions
- PXE runs in WKWebView (iOS) / offscreen document (Chrome) — no Web Workers
- Barretenberg initialized in direct WASM mode (BackendType.Wasm, single-threaded)
- MemoryAztecStore (in-memory KV) instead of IndexedDB — WKWebView crashes on IDB transactions
- PXE snapshot persistence via serialize/deserialize for state recovery across app launches
- Native prover (Swoirenberg) being stabilized — WASM is current fallback

## Aztec SDK
- **Current version:** v4.1.2 (stable, matching testnet)
- SponsoredFPC address is deterministic per SDK version (artifact hash + salt)
- Testnet may not have SponsoredFPC — always verify with `/network-check`
- After SDK version changes: `rm -rf node_modules && npm install --legacy-peer-deps`
- **v4.1.2 breaking changes:**
  - `.send()` / `.simulate()` now return `offchainEffects` + `offchainMessages`
  - Public event pagination: `getPublicEvents` now accepts `{ from, limit }`
  - L2-to-L1 message witness: takes `(message, txHash)` instead of epoch
  - `PublicFeePaymentMethod` and `PrivateFeePaymentMethod` deprecated
- **Testnet RPC:** `https://rpc.testnet.aztec-labs.com/`
- **Mainnet RPC:** `https://rpc.aztec.network/`

## Development Roadmap (3 months — April–June 2026)

### Dual-Track Parallel Strategy
- **Track 1 (iOS/UI):** SDK upgrade → Guardian UI → Bridge UI → DEX UI → TestFlight → App Store
- **Track 2 (Contract/Infra):** Swoirenberg → Bridge contracts → DEX layer → Mainnet → Security

### Phase 1: Foundation (Hafta 1-2)
- SDK v4.1.2 upgrade (all 14 packages + Nargo.toml tags)
- Security fixes: CORS whitelist, postMessage origin, dead code cleanup, version sync
- PXE lazy init + FileManager persistence + incremental sync
- Swoirenberg XCFramework debug (Track 2)

### Phase 2: Guardian + Bridge Contracts (Hafta 3-4)
- celari_recoverable_account v0.2.0 (complete guardian flow)
- Guardian iOS UI (setup + recovery + cancel)
- Bridge L1 contract (Solidity/Forge) + L2 contract (Noir)
- Bridge SDK (bridge-client.ts)

### Phase 3: Bridge UI + WalletConnect (Hafta 5-6)
- Bridge iOS UI (deposit/withdraw flows)
- WalletConnect v2 full integration (deep linking, session persistence)
- DEX contract interaction layer (Track 2)

### Phase 4: DEX + Push + Widget (Hafta 7-8)
- SwapView iOS UI
- Push notifications (local + remote)
- iOS Widget (WidgetKit — balance display)
- Mainnet contract deployment (Track 2)
- E2E integration test suite

### Phase 5: Ship (Hafta 9-12)
- TestFlight beta (week 9-10)
- Security hardening (SSL pinning, jailbreak detection, memory wipe)
- Native prover finalize (or WASM fallback)
- App Store submission (week 11-12)
- Mainnet launch

### Key Decision Gates
- **Week 4:** Swoirenberg stable? → Yes: ship native prover / No: stay WASM
- **Week 10:** TestFlight stable? → Yes: submit App Store / No: fix & re-test
