1. Yonetici Ozeti

Celari Wallet, Aztec Network uzerinde calisan passkey-native bir gizlilik cuzdanidir. P256/secp256r1 imza dogrulamasi yapan ozel bir Noir akilli kontrat ile birlikte iOS native uygulama, Chrome extension, SDK ve tanitim websitesi icermektedir.

Bu denetimde 5 kritik, 8 yuksek, 10+ orta seviye sorun tespit edilmistir. En onemli bulgu, WebAuthn imza formatinin Noir kontrat beklentisiyle uyumsuz olmasidir — bu wallet'in temel islevini etkileyen bir sorundur.

2. Kritik Sorunlar

Kritik 2.1 — WebAuthn Imza Uyumsuzlugu

Dosyalar: contracts/.../main.nr:120-130, src/utils/passkey.ts:128-167, src/utils/passkey_account.ts:41-67

Sorun: Noir kontrat SHA-256(outer_hash_bytes) uzerinden imza dogruluyor. Ancak WebAuthn/Passkey ile imza atildiginda, tarayici aslinda SHA-256(authenticatorData || SHA-256(clientDataJSON)) uzerinden imzaliyor — tamamen farkli bir deger.

Etki: CLI'den raw subtle.sign() ile calisir, ama gercek kullanici tarayicida passkey ile transaction imzaladiginda kontrat REDDEDER. Wallet'in temel vaadi kirilir.

Cozum: (a) Noir kontrati WebAuthn formatini destekleyecek sekilde guncelle — auth witness'a authenticatorData + clientDataJSON ekle, veya (b) tarayici tarafinda raw P256 signing kullan (ama bu passkey amacini bozar).

Kritik 2.2 — Thread-Safety: PXEBridge.pendingCallbacks

Dosya: PXEBridge.swift:16, 134, 142, 152, 308

Sorun: pendingCallbacks dictionary'si hicbir senkronizasyon olmadan birden fazla thread'den erisiyor. withCheckedThrowingContinuation uc farkli yoldan resume edilebilir: (1) JS response, (2) evaluateJavaScript error, (3) timeout. CheckedContinuation iki kez resume edilirse fatal crash olusur.

Cozum: pendingCallbacks erisimini @MainActor ile izole et veya NSLock / actor kullan. Ayrica continuation'i Optional wrapper ile sar, sadece ilk resume'de calistir.

Kritik 2.3 — Private Key UserDefaults'ta

Dosyalar: Account.swift:13-14, WalletStore.swift:562-569, 682-684

Sorun: Account struct'inda secretKey ve privateKeyPkcs8 Codable field olarak mevcut. saveAccounts() cagirildiginda bu degerler UserDefaults'a serialize ediliyor. Keychain bypass edilerek duz metin plist dosyasina yaziliyor.

Etki: Jailbreak'li cihaz, sifresiz iTunes backup, veya forensic extraction ile private key'ler ele gecirilebilir.

Cozum: Account struct'indan secretKey ve privateKeyPkcs8 field'larini kaldir. Bu degerler sadece Keychain'de saklanmali. Mevcut UserDefaults verisinden de temizlenmeli.

Kritik 2.4 — Deploy Server Private Key Donduruyor

Dosya: scripts/deploy-server.ts:115-116

Sorun: /api/deploy endpoint'i P256 private key + Aztec secret key'i HTTP response'da donduruyor. CORS tarayici tarafli; curl ile bypass edilir. Hicbir authentication yok.

Cozum: Private key'i HTTP uzerinden dondurmekten vazgec. Key generation client-side yapilmali. Deploy server sadece public key alip kontrat deploy etmeli.

Kritik 2.5 — JavaScript Injection (String Interpolation)

Dosya: PXEBridge.swift:136

let js = "window._receiveFromSwift('\(jsonString.replacingOccurrences(of: "'", with: "\\'"))')"

Sorun: Sadece tek tirnak escape ediliyor. Backslash iceren JSON, JS string literal'dan breakout yapabilir. Kullanici girdisi (ornegin alici adres) uzerinden arbitrary JavaScript calistirma mumkun.

Cozum: Tum mesaj gonderimlerinde callAsyncJavaScript ile structured arguments kullan (snapshot restore icin zaten yapilmis).

5. Core Katman Detayli Analiz

5.1 Mimari Genel Bakis

Core katman hub-and-spoke mimarisi kullanir. WalletStore merkezi state yoneticisi olarak calisir:

  • WalletStore.swift — 740+ satir, @Observable merkezi state container. Navigation, accounts, tokens, NFTs, WC sessions, network config ve UI state yonetir.
  • PXEBridge.swift — Swift-WKWebView koprusu. Aztec PXE ile cift yonlu JS iletisimi. Request/response pattern (messageId + continuation).
  • PasskeyManager.swift — Actor. WebAuthn passkey olusturma/dogrulama. AuthenticationServices framework kullanimi. Simulator'de mock key fallback.
  • KeychainManager.swift — Static utility. iOS Keychain CRUD. Biometric-protected items (SecAccessControl + .biometryCurrentSet).
  • BackupManager.swift — AES-256-GCM encrypted backup/restore. PBKDF2 key derivation (600K iterations, SHA-256).
  • PXEPersistenceManager.swift — PXE snapshot encrypt/persist. Keychain-stored symmetric key.
  • NetworkManager.swift — Actor. JSON-RPC, connection check, account verification, balance fetching.

5.2 Guclu Yonler

  • Temiz sorumluluk ayrimi (dedicated manager types)
  • Swift concurrency dogru kullanimi (async/await, actor for NetworkManager/PasskeyManager)
  • Privacy-aware logging (os.log Logger, .public annotations)
  • PBKDF2 600K iterations — guncel OWASP onerileriyle uyumlu
  • Idempotent WebView setup
  • Uygun timeout degerleri (ZK proof icin 20 dakikaya kadar)

5.3 Zayif Yonler

  • God Object: WalletStore 740+ satir — navigation, state, persistence, account lifecycle, balance, deploy, demo mode, toast hepsini yonetiyor
  • Tutarsiz thread-safety: NetworkManager/PasskeyManager actor, ama PXEBridge/WalletStore plain class
  • Karisik logging: print() (PasskeyManager) vs Logger
  • Sessiz hata yutma: try? ile hatalar 12+ yerde sessizce yok ediliyor
  • Magic string'ler: "PXE_INIT", "PXE_TRANSFER", "pxeBridge" — enum/constant olmali

7. PXE Bridge ve Polyfill Analizi

7.1 Kopru Mimarisi

iOS uygulamasi PXE'yi katmanli bir WKWebView sistemi uzerinden calistirir:

  1. PXEBridge.swift: 1x1 gizli WKWebView olusturur, script'leri enjekte eder
  2. pxe-bridge-shim.js: Chrome API polyfill'leri (chrome.runtime, chrome.storage, Worker, fetch)
  3. buffer-polyfill.js: Node.js Buffer polyfill (feross/buffer)
  4. pxe-bridge.html: ESM module olarak offscreen.js yukler
  5. offscreen.js: ~69MB minified bundle (Aztec PXE + WASM)

7.2 Shim Sorunlari

  • storageRemove Swift'te handle edilmiyor — callback sonsuza kadar askida
  • chrome.storage.session ile chrome.storage.local ayni UserDefaults — namespace collision
  • Storage callback'ler icin timeout yok — memory leak
  • _receiveFromSwift JSON'u iki kez parse ediyor, ilki try/catch disinda
  • fetch() polyfill tum dosyalar icin application/wasm MIME type donduruyor
  • Worker polyfill hemen error firlatiyor — SDK fallback'ine bagli

7.3 offscreen.js Durumu

iOS Resources'taki offscreen.js dogru iOS build'i (safari17 target, ESM format). Ancak kaynak kodu Chrome-centric pattern'ler iceriyor ve tamamen shim uyumluluguna dayanior.

Yeniden build icin: node extension/build.mjs --ios

8. Noir Kontrat ve Script Analizi

8.1 CelariPasskeyAccount Kontrati

P256/secp256r1 imza dogrulamasi yapan ozel Noir account contract. Aztec'in AccountActions framework'u uzerine insa edilmis.

Kontrat Guvenlik Bulgulari

  • Orta S-value malleability: Her gecerli (r, s) imzasi icin (r, n-s) de gecerlidir. Kontrat bunu kontrol etmiyor.
  • Orta Point-on-curve dogrulama yok: Constructor gecersiz public key'i kabul eder — kullanicinin deploy ucreti bos yere harcanir.
  • Dusuk Auth witness Field truncation: witness[i] as u8 — 255'ten buyuk Field sessizce truncate olabilir.
  • Dusuk #[noinitcheck]: entrypoint ve verify_private_authwit constructor'dan once cagrilabilir.

PublicKeyNote Packable Implementasyonu

Byte ordering dogru: 31 byte big-endian packing, son byte ayri saklanma. BN254 field sinirlari icinde. Ancak fragile — yorum eksik.

8.2 Deploy Script Sorunlari

  • Kritik Private key HTTP response'da donduruluyor
  • Yuksek Deploy/transfer/faucet endpoint'lerinde authentication yok
  • Orta Transfer endpoint aslinda mint_to_public cagrisiyor — from parametresi ignore ediliyor
  • Orta Request body size limiti yok
  • Dusuk Faucet rate limiting memory-only — restart ile sifirlanir
  • Dusuk Bazi scriptlerde hardcoded node URL

9.1 Celari'nin Benzersiz Avantajlari

  1. Aztec'te ilk iOS native wallet — hicbir rakipte yok
  2. Passkey-native mimari — kontrat seviyesinde P256 dogrulama
  3. Secure Enclave — key cikarilmaz, hardware-bound
  4. PXE snapshot persistence — mobilde kritik

9.2 Azguard'dan Ogrenilecekler

  1. Kisi defteri (address book) — dusuk efor, yuksek etki
  2. Otomatik kilit timeout — guvenlik icin sart
  3. Authwit gorunturlugu — kullanici izin farkindligi
  4. Aztec Foundation hibe basvurusu — $100K potansiyeli
  5. Side panel modu — extension UX iyilestirme

9.3 Rekabet Ortami

Aztec wallet ekosistemi hala erken asamada (Subat 2026):

  • Azguard (Baking Bad) — Browser extension, Schnorr-based, en olgun
  • Celari — iOS + extension, passkey-native, custom Noir contract
  • Obsidion (Zpoken) — Passkey + email login, web-based
  • Shinami — aztec-wallet-sdk (gelistirici araci)

Aztec token ($AZTEC) yeni lansman yapti, mainnet aktivitesi basliyor — wallet pazari acik.

10. Oncelikli Aksiyon Plani

Acil — Bu Hafta

  1. WebAuthn imza uyumsuzlugunu coz — Noir kontrati WebAuthn formatini destekleyecek sekilde guncelle veya auth witness'a authenticatorData/clientDataJSON ekle
  2. PXEBridge thread-safety — pendingCallbacks'i @MainActor izolasyonuna al, continuation'i Optional wrapper ile sar
  3. Account struct'indan secretKey/privateKeyPkcs8 sil — bu degerler sadece Keychain'de saklanmali, UserDefaults'tan temizle
  4. evaluateJavaScript → callAsyncJavaScript migration — JS injection onleme
  5. offscreen.js yeniden build — node extension/build.mjs --ios

Kisa Vade — Bu Ay

  1. Transfer/backup oncesi biometric dogrulama ekle
  2. AddAccountView'i duzelt (Keychain + passkey akisi)
  3. WalletStore'u @MainActor yap
  4. Deploy server'dan private key dondurmekten vazgec
  5. Navigation stack implementasyonu (NavigationPath)
  6. ConfirmTxView: activity kaydet + balance refresh
  7. storageRemove handler'i Swift'e ekle

Orta Vade — Bu Ceyrek

  1. Kisi defteri (address book) ekle
  2. Auto-lock timeout implementasyonu
  3. Dynamic Type destegi
  4. Accessibility iyilestirmeleri (labels, traits, VoiceOver)
  5. TLS certificate pinning
  6. S-value malleability kontrolu Noir kontrata ekle
